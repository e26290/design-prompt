- 不要建立文件、演示板、預覽矩陣、輔助組件或裝飾性佈局。除非缺少必要的現有資源且工作無法繼續，否則不要要求做出更多設計決策。
- 建立或搜尋一個名為「Components」的專用頁面。
- 建立一個名為「按鈕」的組件（Component）。

按鈕樣式目標：

成熟、可擴展、設計系統品質的按鈕元件，適用於未來的 React 實作。優先考慮語意標記的使用、清晰的屬性、可維護的結構和規範的自動佈局。

- 語意變數

除非絕對必要，否則請勿建立新的文字樣式。

必需的現有組件：
- 用於前導/尾隨圖示的 `placeholder-icon`
如果找不到其中任何一個，請停止並向我索取確切的節點連結。

## PHASE 3 — 建構規則

---

- 全螢幕使用自動佈局
- 主按鈕外框預設必須緊貼內容
- 按鈕必須保持單行顯示
- 寬度必須隨文字長度自然增長
- 使用基於內邊距的尺寸調整，而非固定寬度
- 每個尺寸都應設定最小高度
- 保持命名和變體組織規範
- 保持頁面簡潔，並面向生產環境

所需結構：
- 按鈕外框預設緊貼內容
- 內部視覺容器填充外部容器，以便手動調整大小能夠正常工作
- 視覺容器內的內容行居中

重要提示：
1. 如果手動加寬按鈕實例，內部視覺容器
2. 也必須正確擴展。
3. 聚焦、加載以及任何內部視覺容器都必須保持正確的
調整大小行為。

## PHASE 6 — 狀態（STATES)

---

States:
- default
- hover
- pressed
- focused
- disabled

Behavior:
- default = 基本視覺效果
- hover = 細微的互動變化
- pressed = 更強的啟動狀態
- focused = 聚焦環 + 預設視覺效果
- disabled = 停用且不交互

已禁用規則：
- 使用已停用的語意表面/文字/圖示/邊框標記
- 無懸停效果
- 無按壓效果

焦點規則：
- 使用可靠的焦點環實現
- 僅在必要時才允許使用包裝器/焦點宿主
- 保持解決方案簡潔且可調整大小

- 可選的尾隨圖標

- 圖示間距必須使用標記
- 此任務中請勿建立僅包含圖示的按鈕

## PHASE 8 — TEXT

---

規則：

- 不要強制大寫
- 文字顏色必須使用與外觀、語氣和狀態相符的語意文字變數
